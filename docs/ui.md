## UI (похоже на Google Calendar)

### Технологии UI

- **HTML/CSS/JavaScript** — статичные файлы, без сборки
- **FullCalendar 6.x** (CDN) — библиотека для представлений календаря и drag-n-drop
- **CSS** — кастомные стили без фреймворков

### Страницы

| Файл | Назначение |
|------|------------|
| `index.html` | Основной интерфейс — календарь с расписаниями |
| `tags.html` | Управление тегами и их значениями |
| `logs.html` | Журнал аудита (история изменений) |
| `reports.html` | Отчёты по расписаниям |
| `clients.html` | Управление клиентами |
| `subscriptions.html` | Управление абонементами (покупки и расходы) |
| `subscription_types.html` | Управление типами абонементов (шаблоны) |

### Основные экраны

- **Календарь**: представления Month / Week / Day
- **Список событий** (альтернативный компактный режим)
- **Мобильная поддержка**: собственная моб. панель навигации и drawer для фильтров

### Фильтры

- **Панель фильтров**: мультивыбор значений тегов
- **Значения тегов** отображаются с цветом (индикатор рядом с чекбоксом)
- **Поиск по заголовку** (опционально)
- **Переключатель интервала**: кнопки «Сегодня», «<», «>», выбор даты
- **Основной тег для подсветки**: выпадающий список — цвет подсветки берётся из цвета значения выбранного тега

### Операции над событиями

- **Создание**: клик/drag на интервал → модальное окно формы
- **Редактирование**: клик по событию → модальное окно
- **Перетаскивание** события (drag & drop) и растягивание длительности (resize)
- **Удаление**: кнопка в модальном окне
- **Отмена**: чекбокс «Отменено» — событие отображается перечёркнутым стилем
- **Копирование**: в модальном окне выпадающий список «Скопировать...» со значениями
  - На завтра / Через неделю / На 2 недели / На 3 недели / На 4 недели
  - Для 2–4 недель создаются серии копий с шагом 1 неделя (7 дней) до выбранного срока
  - Перед созданием копий выполняется пред-проверка пересечений для уникальных ресурсов; при конфликте показана ошибка и копирование отменяется

### Формы

- **Событие**: `title`, `dateFrom`, `dateTo`, выбор нескольких `tagValueIds`, `contact`, чекбокс `isCanceled`
- **Тег**: `name`, чекбоксы `required` и `unique_resource`
- **Значение тега**: `value`, `color` (color picker)

### Поведение и UX

- **Конфликт времени**: API возвращает ошибку при пересечении уникальных ресурсов, UI показывает сообщение
- **Отменённые события**: отображаются перечёркнутым стилем, не учитываются при проверке пересечений
- **Локализация**: русский (ru) для FullCalendar
- **Часовой пояс**: по умолчанию браузера; хранение — ISO-8601 строки
- **Скролл**: внешняя страница без скролла; скролл только внутри календаря; скрыты часы 00:00–07:00; отключён all-day слот

### Журнал аудита (logs.html)

- Отображает историю изменений за последние N дней
- Показывает: время, пользователя, действие, сущность, детали
- Фильтрация по количеству дней

### Управление тегами (tags.html)

- Список всех тегов с флагами required/unique_resource
- Создание/редактирование/удаление тегов
- Для каждого тега — список значений с цветами
- Создание/редактирование/удаление значений тегов
- Color picker для выбора цвета значения

### Управление клиентами (clients.html)

- Список всех клиентов с поиском по имени
- Создание/редактирование/удаление клиентов
- Модальное окно для добавления и редактирования
- Фильтрация по имени

### Абонементы (subscriptions.html)

Основная страница для работы с абонементами клиентов:

- **Выбор клиента**: выпадающий список с поиском по имени
- **Карточка баланса**: крупно отображает остаток занятий, сумму покупок и расходов
- **Два списка**:
  - Покупки (приход): дата, количество занятий, комментарий, срок действия
  - Расходы (занятия): дата, комментарий (каждый расход = 1 занятие)
- **Просроченные абонементы**: выделяются визуально
- **Действия**:
  - Добавить покупку (приход) с выбором шаблона абонемента
  - Добавить расход (занятие)
  - Добавить нового клиента
  - Удалить покупку или расход

### Типы абонементов (subscription_types.html)

Страница для управления шаблонами абонементов:

- Список типов: название, количество занятий, срок действия в днях
- Создание/редактирование/удаление типов
- При добавлении покупки в `subscriptions.html` — выбор шаблона автоматически заполняет поля
