## REST API

Базовый префикс: `/api`

Все эндпоинты предполагают, что доступ к сайту защищён Nginx Basic Auth. Приложение само ролей не вводит.
Имя пользователя передаётся в заголовке `X-Remote-User` и используется для журнала аудита.

---

### Здоровье

**GET** `/api/health`

Проверка работоспособности сервиса.

**Ответ:**
```json
{ "status": "ok" }
```

### Расписания

#### Получить список расписаний

**GET** `/api/schedules`

**Query параметры:**
| Параметр | Тип | Описание |
|----------|-----|----------|
| from | string (ISO-8601) | Начало интервала фильтрации |
| to | string (ISO-8601) | Конец интервала фильтрации |
| tag_value_ids | string | ID значений тегов через запятую (1,2,3) |

**Логика фильтрации:**
- Пересечение по времени — полуоткрытый интервал `[from, to)`
- Фильтрация по тегам: **И** между группами тегов, **ИЛИ** внутри группы

**Ответ:**
```json
[
  {
    "id": 1,
    "title": "Тренировка",
    "dateFrom": "2025-01-15T10:00:00",
    "dateTo": "2025-01-15T11:30:00",
    "tagValueIds": [1, 3],
    "isCanceled": false,
    "contact": "+7 999 123-45-67"
  }
]
```

#### Создать расписание

**POST** `/api/schedules`

**Тело запроса:**
```json
{
  "title": "Тренировка",
  "dateFrom": "2025-01-15T10:00:00",
  "dateTo": "2025-01-15T11:30:00",
  "tagValueIds": [1, 3],
  "contact": "+7 999 123-45-67"
}
```

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| title | string | Да | Название события |
| dateFrom | string (ISO-8601) | Да | Начало события |
| dateTo | string (ISO-8601) | Да | Конец события |
| tagValueIds | number[] | Нет | ID значений тегов |
| contact | string | Нет | Контактная информация |

**Валидация:**
- `dateTo >= dateFrom`
- Для **required** тегов — должно быть выбрано хотя бы одно значение
- Для **unique_resource** тегов — проверка пересечений по времени с существующими событиями

**Ответ:** Созданный объект расписания с присвоенным `id`

**Ошибки:**
- `400` — Некорректные данные, отсутствуют required теги, пересечение с уникальным ресурсом

#### Обновить расписание

**PUT** `/api/schedules/{id}`

**Тело запроса:**
```json
{
  "title": "Новое название",
  "dateFrom": "2025-01-15T10:00:00",
  "dateTo": "2025-01-15T12:00:00",
  "tagValueIds": [1, 2],
  "isCanceled": false,
  "contact": "+7 999 000-00-00"
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| title | string | Название события |
| dateFrom | string (ISO-8601) | Начало события |
| dateTo | string (ISO-8601) | Конец события |
| tagValueIds | number[] | ID значений тегов |
| isCanceled | boolean | Признак отмены события |
| contact | string | Контактная информация |

Все поля опциональны — передаются только изменяемые.

**Ответ:** Обновлённый объект расписания

**Ошибки:**
- `404` — Расписание не найдено
- `400` — Некорректные данные

#### Удалить расписание

**DELETE** `/api/schedules/{id}`

**Ответ:** `204 No Content`

**Ошибки:**
- `404` — Расписание не найдено

### Теги

#### Получить список тегов

**GET** `/api/tags`

**Ответ:**
```json
[
  {
    "id": 1,
    "name": "зал",
    "required": true,
    "unique_resource": true
  },
  {
    "id": 2,
    "name": "тренер",
    "required": false,
    "unique_resource": false
  }
]
```

#### Создать тег

**POST** `/api/tags`

**Тело запроса:**
```json
{
  "name": "зал",
  "required": true,
  "unique_resource": true
}
```

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| name | string | Да | Уникальное имя тега |
| required | boolean | Нет | Обязательный тег (default: false) |
| unique_resource | boolean | Нет | Уникальный ресурс (default: false) |

**Ответ:** Созданный объект тега

**Ошибки:**
- `400` — Тег с таким именем уже существует

#### Обновить тег

**PUT** `/api/tags/{id}`

**Тело запроса:**
```json
{
  "name": "новое имя",
  "required": false,
  "unique_resource": true
}
```

**Ответ:** Обновлённый объект тега

**Ошибки:**
- `404` — Тег не найден
- `400` — Тег с таким именем уже существует

#### Удалить тег

**DELETE** `/api/tags/{id}`

Удаляет тег и все связанные значения (каскадное удаление).

**Ответ:** `204 No Content`

**Ошибки:**
- `404` — Тег не найден

### Значения тегов

#### Получить значения тега

**GET** `/api/tags/{tagId}/values`

**Ответ:**
```json
[
  {
    "id": 1,
    "tag_id": 1,
    "value": "зал1",
    "color": "#FF5733"
  },
  {
    "id": 2,
    "tag_id": 1,
    "value": "зал2",
    "color": "#33FF57"
  }
]
```

#### Создать значение тега

**POST** `/api/tags/{tagId}/values`

**Тело запроса:**
```json
{
  "value": "зал3",
  "color": "#3357FF"
}
```

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| value | string | Да | Значение тега |
| color | string | Нет | HEX цвет (#RRGGBB) |

**Ответ:** Созданный объект значения тега

**Ошибки:**
- `404` — Тег не найден
- `400` — Значение уже существует для этого тега

#### Обновить значение тега

**PUT** `/api/tags/values/{id}`

**Тело запроса:**
```json
{
  "value": "зал3 (новый)",
  "color": "#AABBCC"
}
```

**Ответ:** Обновлённый объект значения тега

**Ошибки:**
- `404` — Значение тега не найдено
- `400` — Такое значение уже существует для этого тега

#### Удалить значение тега

**DELETE** `/api/tags/values/{id}`

**Ответ:** `204 No Content`

**Ошибки:**
- `404` — Значение тега не найдено

### Журнал аудита

#### Получить записи аудита

**GET** `/api/audit`

**Query параметры:**
| Параметр | Тип | Default | Описание |
|----------|-----|---------|----------|
| days | integer | 2 | Количество дней истории (1-365) |

**Ответ:**
```json
[
  {
    "id": 42,
    "ts": "2025-01-15T10:30:00Z",
    "username": "admin",
    "action": "UPDATE",
    "entity": "schedules",
    "entityId": 15,
    "details": "title: Старое -> Новое; date_from: 2025-01-15T09:00 -> 2025-01-15T10:00"
  },
  {
    "id": 41,
    "ts": "2025-01-15T09:15:00Z",
    "username": "admin",
    "action": "CREATE",
    "entity": "schedules",
    "entityId": 15,
    "details": "title=Тренировка; from=2025-01-15T09:00; to=2025-01-15T10:00"
  }
]
```

| Поле | Тип | Описание |
|------|-----|----------|
| id | integer | Уникальный ID записи |
| ts | string (ISO-8601 UTC) | Время операции |
| username | string \| null | Имя пользователя |
| action | string | Тип действия: CREATE, UPDATE, DELETE |
| entity | string | Имя сущности: schedules, tags, tag_values |
| entityId | integer \| null | ID изменённой записи |
| details | string \| null | Детали изменений |

---

### Форматы данных

- **Время** передаётся и возвращается в ISO-8601 строках (например, `2025-01-15T10:00:00+03:00`)
- **Пересечение по времени** — полуоткрытый интервал `[from, to)`
- **Параметр `tag_value_ids`**: значения группируются по `tagId`; событие проходит фильтр, если содержит хотя бы одно значение из каждой выбранной группы

### Бизнес-логика

При создании/редактировании события:
1. **Required теги** — должно быть выбрано хотя бы одно значение каждого обязательного тега
2. **Unique resource теги** — проверяются пересечения по времени с другими неотменёнными событиями; при пересечении — ошибка 400 с описанием конфликта
3. **Отменённые события** (`isCanceled=true`) не учитываются при проверке пересечений

### Клиенты

#### Получить список клиентов

**GET** `/api/clients`

**Query параметры:**
| Параметр | Тип | Описание |
|----------|-----|----------|
| search | string | Поиск по имени (частичное совпадение) |

**Ответ:**
```json
[
  {
    "id": 1,
    "name": "Иванов Иван"
  },
  {
    "id": 2,
    "name": "Петров Пётр"
  }
]
```

#### Получить клиента по ID

**GET** `/api/clients/{id}`

**Ответ:**
```json
{
  "id": 1,
  "name": "Иванов Иван"
}
```

**Ошибки:**
- `404` — Клиент не найден

#### Создать клиента

**POST** `/api/clients`

**Тело запроса:**
```json
{
  "name": "Сидоров Сидор"
}
```

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| name | string | Да | Имя клиента |

**Ответ:** Созданный объект клиента с присвоенным `id`

#### Обновить клиента

**PUT** `/api/clients/{id}`

**Тело запроса:**
```json
{
  "name": "Новое имя"
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| name | string | Имя клиента |

**Ответ:** Обновлённый объект клиента

**Ошибки:**
- `404` — Клиент не найден

#### Удалить клиента

**DELETE** `/api/clients/{id}`

**Ответ:** `204 No Content`

**Ошибки:**
- `404` — Клиент не найден

---

### HTTP коды ответов

| Код | Описание |
|-----|----------|
| 200 | Успешный запрос (GET, PUT) |
| 201 | Создание успешно (POST) — в FastAPI возвращает 200 |
| 204 | Удаление успешно (DELETE) |
| 400 | Некорректные данные |
| 404 | Ресурс не найден |
| 500 | Внутренняя ошибка сервера |
